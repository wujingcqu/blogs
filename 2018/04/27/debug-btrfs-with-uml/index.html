debug_btrfs_with_uml

In this post, I will explain how to debug/understand the working flow of btrfs with User Mode Linux (UML).
After lots of trial and errors, I find that it is pretty convenient to use the UML and a customized root file system generated by buildroot.

I will list the majors steps in this post.

Building UML Kernel

Prepare a clean kernel source, if you have executed make aginst the current architecture, such as x86, you should execute make clean && make distclean.

Select the btrfs and complile it as built-in, (File systems -> Btrfs filesystem)

Select the ramdisk driver, we use ramdisk to mount btrfs. (Block devices -> RAM block device support)
Set the number of RAM disk to 1, the default size to 1048576 (1G), to make it big enough to test btrfs.
As follows:

Build the kernel

The UML kernel builds very fast, the whole process takes about 1 miniute on my working server.

The resulting vmlinux images contains the required debug symbols and is directly runnalbe from command line.

You can also refer to the following btrfs wiki for kernel building.

Prepare root file system for UML

The linux kernel cannot boot without a root file system. Thus, we need to feed the vmlinux kernel with a root file system.
One of the requirement of root file system is that it must contain user-space tools to manipulate btrfs, e.g., btrfs-prog.

Fortunately, the buildroot packages provide the necessary tools to build a running root file system
with the necessary programs. I do not plan to give detailed instructions on how to use buildroot, instead, I will post the
working root file system in this post.
